2 # Towards Autonomy Perception Library (TAPL)
3 
4 Goal of this library is to provide an easy and quick way of implementing perception pipelines.

10 ## Examples of Perception Task
11 
12 #### Visual Odometry for a sequence of Monocular camera images
13 
14 This example is provided at ```examples/src/monoVO.cpp```. It uses a sequence of monocular images to perform visual odometry and build sparse point-cloud. This functionality is provided as an API and can be accessed using the function: ```tapl::cve::computeRelativePose()```.
15 
16 **Pipeline**
17 
18  - Read images and push into a ring buffer.
19  - If more than one image is available in the buffer then:
20  - Perform keypoint detection and matching.
21  - Compute essential matrix.
22  - Compute relative pose (R, t)
23  - Triangulate good keypoints for which a match is found.
24  - Compute global pose from this relative pose.

29 #### LiDAR Object Detection
30 
31 This example is provided at ```examples/src/lidarObjectDetection.cpp```. It reads in PCD point-cloud files, performs downsampling, ground-plane segmentation, and clustering, and then some post-processing filtering to get 3D Bounding-Box of objects.
32 
33 **Pipeline**
34 
35  - Load point-cloud data.
36  - Downsample point-cloud (voxelization).
37  - Crop the point-cloud based on a region of interest.
38  - Segment out ground-plane using RANSAC.
39  - For *n* iterations:
40  - Choose 3 random points.
41  - Fit a plane using least-squares.
42  - Count number of inliers within a certain distance threshold between each point and plane.
43  - Choose the plane that resulted in maximum number of inliers.
44  - Implemented as part of ```class tapl::pte::Plane()```.
45  - Perform Euclidian Clustering within the segmented point-cloud.
46  - Store the point-cloud as a **KdTree**. Implemented as ```struct tapl::pte::KdTree```.
47  - Perform euclidean clustering on the KdTree. Implemented as ```class tapl::pte::EuclideanCluster()```.
48  - Filter and Extract the bounding-boxes.
